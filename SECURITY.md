# Security Policy

Si encuentras un problema de seguridad en SAVK, no lo publiques primero en un
issue abierto.

Mientras el repositorio no tenga un canal dedicado, la forma recomendada es:

- contactar al mantenedor en privado
- incluir pasos de reproducción
- incluir impacto esperado
- incluir versión o commit afectado

## Scope

Reporta por esta vía si el problema implica:

- exposición de secretos en reportes
- bypass de redacción de evidencia
- ejecución inesperada de comandos
- manejo inseguro de artefactos o contratos

Problemas de calidad, UX o bugs normales pueden abrirse como issues públicos.
